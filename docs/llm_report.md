# Отчёт об ознакомлении с большими языковыми моделями

## Введение

В рамках проектной практики был изучен базовый принцип работы современных больших языковых моделей. Отдельное внимание было уделено применению LLM при подготовке документации, создании структуры репозитория, оформлении сайта и разработке вариативной части.

В качестве основного учебного источника использовался интерактивный разбор с **ChatGPT 5.5 in Codex**. Такой формат позволил не только получить теоретические пояснения, но и сразу применить их к текущей практической задаче.

## Изученные темы

### Большие языковые модели

Большие языковые модели, или LLM, предназначены для обработки и генерации текста на естественном языке. Они могут отвечать на вопросы, объяснять темы, помогать писать документацию, анализировать код и предлагать варианты решения задач.

Ключевая идея работы LLM заключается в том, что модель получает текстовый запрос и на основе обученных закономерностей формирует наиболее вероятное продолжение ответа. При этом модель не является обычной базой данных: она генерирует ответ на основе контекста, а не просто копирует готовую запись.

### Архитектура Transformer

Современные LLM основаны на архитектуре Transformer. Её важная особенность - механизм внимания, который позволяет модели учитывать связи между разными частями текста. Благодаря этому модель может работать с длинными запросами, удерживать контекст и связывать между собой разные фрагменты информации.

В рамках практики было важно понять не математические детали архитектуры, а общий принцип: модель анализирует входной текст, выделяет значимые связи и на их основе формирует ответ.

### Промпт-инжиниринг

Промпт-инжиниринг - это способ составления запросов к языковой модели так, чтобы получить более точный и полезный результат.

В ходе работы использовались следующие приёмы:

- формулирование задачи короткими шагами;
- уточнение ограничений перед генерацией текста;
- просьба не выполнять лишние действия;
- проверка результата по исходному заданию;
- разбиение большой задачи на отдельные этапы.

Например, вместо общего запроса “сделай практику” эффективнее использовать запросы вида:

```text
Сделай только пункт 1.2: Markdown-документацию в docs.
Не пушь изменения, сначала дай посмотреть.
```

Такой подход снижает риск лишних изменений и помогает контролировать результат.

### Этические аспекты использования ИИ

При использовании LLM важно учитывать ограничения и риски:

- модель может ошибаться или уверенно формулировать неточные ответы;
- сгенерированный текст нужно проверять по заданию;
- нельзя выдавать автоматически созданный материал за полностью самостоятельный анализ без проверки;
- не следует передавать модели приватные данные, пароли и токены;
- результат должен быть адаптирован под реальную ситуацию студента.

В рамках практики LLM использовалась как помощник для оформления, структурирования и проверки материалов. Итоговые файлы были дополнительно просмотрены и сопоставлены с требованиями задания.

## Применение LLM при выполнении практики

LLM применялась в нескольких частях работы.

### Анализ задания

Сначала с помощью LLM были разобраны требования задания. Это помогло выделить основные части практики:

- обязательная часть;
- Markdown-документация;
- сайт;
- отчёт;
- вариативная часть.

После этого стало понятно, что вариативная часть с HTTP-сервером не должна заменять всю практику, а должна быть отдельным разделом.

### Подготовка документации

LLM помогла составить структуру документов в папке `docs`:

- `project.md`;
- `members.md`;
- `journal.md`;
- `resources.md`;
- `llm_report.md`;
- `variant_guide.md`.

Также была предложена логика содержания: писать не о стороннем проекте, а именно о проектной практике, выполняемой в рамках текущего задания.

### Создание сайта

При создании сайта LLM использовалась для подбора структуры HTML-страницы. В результате сайт получил разделы:

- главная страница;
- о практике;
- участник;
- журнал;
- вариативная часть;
- ресурсы.

Дополнительно была добавлена простая схема, показывающая связь между Git, Markdown, HTML/CSS, Python и итоговым отчётом.

### Работа с кодом

LLM использовалась при реализации и объяснении учебного HTTP-сервера на Python. Она помогла разбить работу сервера на понятные этапы:

1. запуск сокета;
2. ожидание подключения;
3. получение HTTP-запроса;
4. поиск файла;
5. формирование HTTP-ответа;
6. отправка ответа браузеру.

Такой разбор помог лучше понять, что делает программа и как объяснить её работу на защите.

## Полученные знания

В результате ознакомления с LLM были получены следующие знания:

- LLM можно использовать как инструмент для анализа задания и планирования работы;
- качество ответа сильно зависит от точности запроса;
- сложные задачи лучше разбивать на небольшие шаги;
- сгенерированный текст нужно проверять и редактировать;
- LLM полезна при подготовке документации, но не заменяет понимание предметной области;
- при работе с кодом важно запускать проверки, а не доверять только текстовому объяснению.

## Вывод

Ознакомление с большими языковыми моделями показало, что LLM может быть полезным инструментом при выполнении учебной практики. Она помогает быстрее структурировать материалы, находить ошибки, формулировать документацию и объяснять код.

При этом использование LLM требует контроля: необходимо проверять результат, сверять его с заданием и не переносить в работу неподтверждённые сведения. В данной практике LLM применялась как помощник для организации работы и подготовки материалов, а итоговые файлы были дополнительно проверены перед размещением в репозитории.
